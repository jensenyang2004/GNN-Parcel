# PARCEL — Partitioned Attention-based Reverse Curricula for Enhanced Learning

Implementation of PARCEL from the AAAI-26 paper:

> **Spatially Grouped Curriculum Learning for Multi-Agent Path Finding**
> Thomy Phan, Sven Koenig — AAAI 2026

Built on top of [rl4mapf](https://github.com/thomyphan/rl4mapf) (the CACTUS codebase by the same author).

---

## How PARCEL Works

PARCEL trains multiple agents to find conflict-free paths by combining three ideas:

**1. Spatial Grouping** — At the start of each episode, agents whose bounding regions overlap are grouped together. Agents in different groups are trained independently (no coordination needed if they can't collide).

**2. Masked Self-Attention Critic** — Within each group, a self-attention critic lets agents coordinate during training. The grouping mask ensures agents in different groups are invisible to each other.

**3. Reverse Curriculum (CACTUS)** — Training starts with small bounding regions (easy, short-range goals). As agents improve, the regions grow until all agents eventually merge into one fully coordinated group.

```
Early training:   [A]  [B]  [C]   → 3 independent groups, small goals
Mid training:     [A, B]  [C]     → partial coordination, medium goals  
Late training:    [A, B, C]       → full coordination, goals anywhere
```

After training, only the **actor** (decentralized policy) is used for execution — no communication or replanning needed.

## Quick Start

### Step 1 — Generate benchmark maps (run once)

```bash
python generate_maps.py
```

This creates two map files in `instances/`:
- `random-64-64-10.map` — 64×64 grid with ~10% random obstacles
- `warehouse.map` — structured warehouse with shelves and corridors

### Step 2 — Train PARCEL

**16 agents (paper Section 5.2):**
```bash
python train_parcel.py --nr_agents 16 --epochs 2000
```

**64 agents:**
```bash
python train_parcel.py --nr_agents 64 --epochs 2000
```

Training output is saved to `output/parcel/` by default.

## Training Arguments

| Argument | Default | Paper value | Description |
|---|---|---|---|
| `--nr_agents` | 16 | 16 or 64 | Number of agents |
| `--epochs` | 2000 | 2000 | Training epochs (W) |
| `--episodes_per_epoch` | 10 | 10 | Episodes per epoch (Y) |
| `--time_limit` | 256 | 256 | Max steps per episode (T) |
| `--improvement_threshold` | 0.5 | 0.5 | Curriculum threshold U |
| `--deviation_factor` | 2.0 | 2.0 | Curriculum deviation factor η |
| `--sliding_window` | 50 | 50 | Window size for curriculum |
| `--hidden_dim` | 64 | 64 | Actor hidden layer size |
| `--embed_dim` | 64 | 64 | Attention embedding dim z |
| `--nr_heads` | 2 | 2 | Number of attention heads |
| `--lr` | 0.001 | 0.001 | Adam learning rate |
| `--crop_size` | 64 | 64 | Sub-grid crop size |
| `--output_dir` | output/parcel | — | Where to save model |
| `--seed` | 42 | — | Random seed |
| `--verbose` | off | — | Print grouping info per episode |

## Training Output

Each epoch logs:

```
Epoch  700 | Ralloc=  7 | Train CR=0.781 | Test CR=0.703 | Test SR=0.250 | Time=186.5s
```

| Field | Meaning |
|---|---|
| `Ralloc` | Current allocation radius (grows as policy improves) |
| `Train CR` | Completion rate on training episodes this epoch |
| `Test CR` | Completion rate on test episodes (full map, no radius limit) |
| `Test SR` | Success rate — fraction of episodes where **all** agents reached their goals |
| `Time` | Wall time for the last log interval |

**CR (Completion Rate)** = fraction of agents that reached their goal.
**SR (Success Rate)** = fraction of episodes where every single agent succeeded (stricter).

Model checkpoints are saved every 500 epochs and at the end of training.

## Expected Results (Paper Section 5.2)

Training progress on Random map, 16 agents:

| Episodes | Ralloc | Approx. Test CR |
|---|---|---|
| ~0 | 2 | 0.0 |
| ~6,000 | 2–4 | 0.5–0.7 |
| ~10,000 | 7–15 | 0.7–0.9 |
| ~20,000 | large | ~0.95 |

The curriculum upgrades Ralloc when the policy consistently achieves
`µ - 2σ ≥ 0.5` over the last 50 epochs (≈97% confidence that true CR ≥ 50%).

## How the Curriculum Works

```
Ralloc = 2: goal placed within 2 steps of start → easy, independent agents

     ↓ once CR > 50% with high confidence

Ralloc = 3: goal placed within 3 steps → slightly harder, some agents start grouping

     ↓ ...keeps growing...

Ralloc = 64: goal can be anywhere in the 64x64 map → full coordination required
```

Each training episode uses a randomly cropped 64×64 sub-grid from either the
Random or Warehouse map, then randomly rotates and mirrors it — this gives the
policy diversity to generalize to unseen maps.
